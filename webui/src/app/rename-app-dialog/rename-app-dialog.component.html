Assign a friendly name to the app. The name may include whitespace and does not have to use the default [app-type]@[machine-address] notation. However, when using this notation, the [machine-address] must be an address or FQDN of a machine monitored in Stork; otherwise, an error will be returned. To avoid the error, use a special notation with a double @ character, e.g. dhcp@@machineX. In that case, machineX does not have to exist in the system.
